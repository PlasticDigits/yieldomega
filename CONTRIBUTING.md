# Contributing

## License

Default license for original work in this repository is **GNU Affero General Public License v3.0** (AGPL-3.0). The full license text is in [LICENSE](LICENSE). By contributing, you agree that your contributions are licensed under AGPL-3.0 unless you and the maintainers explicitly agree otherwise in writing for specific material.

## Copyright and attribution

Follow the copyright and SPDX-style notices used in nearby files. New files should include an appropriate copyright line and license identifier consistent with the rest of the tree.

## Licensing of new and changed files

- New **original** source files: AGPL-3.0 unless explicitly marked otherwise and **approved by maintainers**.
- **Patches** to AGPL-licensed files: AGPL-3.0.

Exceptions must be clearly labeled in the affected files and in the change description.

## Third-party code

Do not commit code whose license is incompatible with how this project distributes combined works. Document third-party additions as the project adopts a `NOTICE` file or `third-party/` listing (see [docs/licensing.md](docs/licensing.md)). Respect upstream license terms for dependencies (Foundry, Rust crates, npm packages, etc.).

## Network services and AGPL

If you **modify** AGPL-covered components (for example the indexer or a network-facing frontend) and **expose** that modified version so users interact over a network, you must meet **Corresponding Source** obligations under AGPL-3.0, including the remote-network terms in section 13 of the license. See [docs/licensing.md](docs/licensing.md).

Project documentation, automation, and agent-oriented content must **not** encourage circumventing these obligations.

## Legal assurance (lightweight)

By submitting a contribution, you represent that you have the right to license your contribution under the terms stated here and that it does not knowingly violate third-party rights. If the project later adopts a **Developer Certificate of Origin (DCO)** or **Contributor License Agreement (CLA)**, [docs/licensing.md](docs/licensing.md) will link to it and additional sign-off steps may apply.

## Process

Use issues and pull requests as the project adopts them. Describe what changed and why; link related issues when relevant. For questions about licensing or compliance, prefer maintainers and [docs/licensing.md](docs/licensing.md).
